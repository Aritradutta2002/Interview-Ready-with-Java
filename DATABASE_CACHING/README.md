# Database & Caching

## Overview

This area covers database technologies and caching strategies for Java backend applications. Topics include SQL databases (PostgreSQL, MySQL), NoSQL databases (MongoDB, Cassandra, DynamoDB), query optimization, caching strategies (Redis, Memcached), and connection pooling.

## Interview Tips by Experience Level

### Junior (0-2 years)
- Focus: SQL basics, CRUD operations, basic indexing
- Depth: Write queries, understand primary/foreign keys
- Key Topics: SELECT, JOIN, WHERE, indexes, transactions

### Mid-Level (2-5 years)
- Focus: Query optimization, NoSQL basics, caching patterns
- Depth: Optimize queries, choose right database, implement caching
- Key Topics: Explain plans, N+1 problem, Redis, cache patterns

### Senior (5+ years)
- Focus: Database architecture, scaling strategies, consistency trade-offs
- Depth: Design data models, handle high scale, troubleshoot performance
- Key Topics: Sharding, replication, CAP theorem, distributed transactions

## Most Asked Topics

1. **SQL Fundamentals** - Joins, subqueries, aggregations, window functions
2. **Indexing** - B-tree indexes, composite indexes, covering indexes
3. **Transactions** - ACID properties, isolation levels, deadlocks
4. **Query Optimization** - Explain plans, N+1 problem, query tuning
5. **NoSQL Databases** - Document stores, key-value stores, CAP theorem
6. **Caching Strategies** - Cache-aside, write-through, write-behind, TTL
7. **Redis** - Data structures, persistence, pub/sub, use cases

## Study Strategy

1. Master SQL fundamentals and join types
2. Learn indexing strategies and query optimization
3. Understand ACID properties and transaction isolation
4. Study NoSQL databases and when to use them
5. Learn caching patterns and Redis
6. Understand connection pooling configuration
7. Practice real-world database design scenarios
