## Interview Tips by Experience Level

### Junior (0-2 years)
- Focus: Basic AWS services, Docker basics, CI/CD concepts
- Depth: Understand what services do and when to use them
- Key Topics: EC2, S3, RDS, basic Docker commands

### Mid-Level (2-5 years)
- Focus: Kubernetes, CI/CD pipelines, monitoring setup
- Depth: Hands-on experience deploying and managing applications
- Key Topics: K8s deployments, Jenkins pipelines, CloudWatch/Prometheus

### Senior (5+ years)
- Focus: Architecture decisions, cost optimization, incident response
- Depth: Design production systems, troubleshoot complex issues
- Key Topics: Multi-region deployment, disaster recovery, performance tuning

## Most Asked Topics

1. **AWS Services** - EC2, S3, RDS, Lambda, API Gateway
2. **Kubernetes** - Pods, Services, Deployments, ConfigMaps, Secrets
3. **CI/CD Pipelines** - Jenkins, GitHub Actions, deployment strategies
4. **Monitoring** - Prometheus, Grafana, CloudWatch, metrics and alerts
5. **Logging** - ELK Stack, centralized logging, log analysis
6. **Infrastructure as Code** - Terraform, CloudFormation
7. **Production Issues** - Debugging, incident response, performance tuning

## Study Strategy

1. Start with AWS fundamentals and core services
2. Learn Kubernetes for container orchestration
3. Practice building CI/CD pipelines
4. Study monitoring and logging tools
5. Learn infrastructure as code with Terraform
6. Understand production engineering practices
7. Practice troubleshooting production scenarios
